# Novelty-audit corpus (65 works, recon phase) — MASFuzzer

- **AgentVigil / AgentFuzzer: Generic Black-Box Red-teaming for Indirect Prompt Injection against LLM Agents** (EMNLP Findings 2025 / arXiv 2505.05849)
  - does: Black-box fuzzing of single LLM agents for indirect prompt injection; builds an initial seed corpus, then uses Monte Carlo Tree Search (MCTS)-based seed selection and genetic mutation to iteratively refine adversarial payloads; optimizes attack success rate across tasks; evaluated on AgentDojo (71% ASR) and VWA-adv (70% ASR); does NOT address multi-agent topologies or coverage-guided behavioral exploration.
  - vs MASFuzzer: C1 (fuzzer for LLM agents) — directly overlaps; MASFuzzer must differentiate by targeting multi-agent interaction boundaries, role delegation, and tool-chain composition rather than single-agent indirect injection.
  - https://arxiv.org/abs/2505.05849
- **FLARE: Agentic Coverage-Guided Fuzzing for LLM-Based Multi-Agent Systems** (arXiv 2604.05289, April 2026)
  - does: White-box (source-code-level) coverage-guided fuzzer for multi-agent LLM systems; extracts agent specifications and behavioral spaces from source code, builds test oracles, and conducts coverage-guided fuzzing to expose failures (infinite loops, failed tool invocations); achieves 96.9% inter-agent and 91.1% intra-agent coverage, finding 56 previously unknown failures across 16 open-source apps; focuses on reliability/correctness failures, NOT adversarial security exploitation.
  - vs MASFuzzer: C1 (fuzzer) + C2 (continuum of faults) — closest white-box multi-agent competitor; MASFuzzer must show its black-box adversarial orientation and certainty-as-fitness signal are complementary/superior to FLARE's coverage metric.
  - https://arxiv.org/abs/2604.05289
- **ChainFuzzer: Greybox Fuzzing for Workflow-Level Multi-Tool Vulnerabilities in LLM Agents** (arXiv 2603.12614, March 2026)
  - does: Greybox fuzzer targeting multi-tool dataflow vulnerabilities in single LLM agent apps (not multi-agent); identifies high-impact sink operations, extracts cross-tool dependency chains (96.49% precision), uses Trace-guided Prompt Solving (TPS) to synthesize stable triggering prompts, and performs guardrail-aware payload mutation; confirmed 365 unique reproducible vulnerabilities across 19/20 apps on 998 tools; does NOT model inter-agent communication or role delegation.
  - vs MASFuzzer: C1 (fuzzer) + C5 (real frameworks) — covers multi-tool chaining but within a single agent; MASFuzzer's accountability-sink claim (C4) and multi-agent delegation are the key differentiators.
  - https://arxiv.org/abs/2603.12614
- **T-MAP: Red-Teaming LLM Agents with Trajectory-aware Evolutionary Search** (arXiv 2603.22341, March 2026)
  - does: Black-box evolutionary red-teaming of single LLM agents (including MCP-tool ecosystems); uses agent execution trajectories to guide evolutionary search for adversarial prompts that bypass guardrails AND realize harmful objectives through actual tool interactions; achieves average Attack Realization Rate (ARR) of 57.8% against frontier models (GPT-5.2, Gemini-3-Pro); does NOT address multi-agent manager/worker architectures or certainty signals.
  - vs MASFuzzer: C1 (fuzzer) + C6 (independent oracle via ARR) — trajectory-level oracle is closest analog to MASFuzzer's certainty-as-fitness; MASFuzzer must show that certainty in multi-agent delegation (C3/C4) is a distinct fitness signal beyond trajectory-level harm realization.
  - https://arxiv.org/abs/2603.22341
- **AgentDojo: A Dynamic Environment to Evaluate Prompt Injection Attacks and Defenses for LLM Agents** (NeurIPS 2024)
  - does: Benchmark/evaluation environment (not a fuzzer) for prompt injection against single LLM agents across 5 task domains (workspace, banking, travel); provides 97 user tasks and 629 security test cases with dual metrics (ASR and utility-under-attack); attack and defense methods are tested manually or via provided baselines; does NOT automate attack generation or provide coverage guidance.
  - vs MASFuzzer: C5 (real frameworks) + C6 (independent oracle) — widely used as the evaluation harness by AgentVigil and others; MASFuzzer's multi-agent extension would need to justify why AgentDojo's single-agent oracle is insufficient.
  - https://www.researchgate.net/publication/397198170_AgentDojo_A_Dynamic_Environment_to_Evaluate_Prompt_Injection_Attacks_and_Defenses_for_LLM_Agents
- **AgentPoison: Red-teaming LLM Agents via Poisoning Memory or Knowledge Bases** (NeurIPS 2024 / arXiv 2407.12784)
  - does: Black-box backdoor attack (not fuzzer) against RAG-based and memory-augmented LLM agents; optimizes a universal backdoor trigger that poisons a small fraction (<0.1%) of the knowledge base to reliably retrieve malicious instructions; no model fine-tuning required; >80% ASR across driving, QA, and healthcare agents; targets memory/retrieval layer, not tool-call chains or multi-agent delegation.
  - vs MASFuzzer: C1 (automated adversarial agent testing) — distinct attack surface (memory poisoning vs. tool-chain injection); MASFuzzer's focus on live inter-agent communication and certainty-as-fitness is not addressed here.
  - https://arxiv.org/abs/2407.12784
- **In-Browser LLM-Guided Fuzzing for Real-Time Prompt Injection Testing in Agentic AI Browsers** (arXiv 2510.13543, October 2025)
  - does: Black-box in-browser fuzzer for prompt injection in browser-embedded LLM agents; generates malicious webpage content from templates or via on-the-fly LLM mutation, feeds to browser agent, and uses real-time feedback (DOM context + action monitoring) to evolve attacks; zero false positives; by iteration 10, even top-performing tools fail 58-74% of tests; scope is single browser agent, not multi-agent pipelines.
  - vs MASFuzzer: C1 (fuzzer) + C5 (real frameworks, browser agents) — demonstrates iterative LLM-guided mutation is practical; MASFuzzer's certainty-as-fitness (C3) and multi-agent accountability-sink (C4) are not addressed.
  - https://arxiv.org/abs/2510.13543
- **Who Tests the Testers? Systematic Enumeration and Coverage Audit of LLM Agent Tool Call Safety (SafeAudit)** (arXiv 2603.18245, March 2026)
  - does: White-box meta-audit framework (SafeAudit) that uses an LLM-based enumerator to systematically generate test cases by enumerating valid tool-call workflows and user scenarios; introduces 'rule-resistance' as a non-semantic coverage metric to identify unsafe interaction patterns missed by existing benchmarks; finds >20% residual unsafe behaviors across 3 benchmarks and 12 environments; audits existing test suite completeness rather than fuzzing for novel attacks.
  - vs MASFuzzer: C6 (independent oracle) + C2 (continuum of coverage) — directly addresses oracle completeness and coverage gaps; MASFuzzer's certainty-as-fitness (C3) is a complementary fitness signal to rule-resistance, but must be explicitly differentiated.
  - https://arxiv.org/pdf/2603.18245
- **RvB: Automating AI System Hardening via Iterative Red-Blue Games** (arXiv 2601.19726, January 2026)
  - does: Training-free iterative red-blue game framework where a Red Team LLM autonomously exposes vulnerabilities (CVEs, jailbreaks) and a Blue Team LLM learns defenses without parameter updates; validated on code hardening and guardrail optimization achieving 90% and 45% Defense Success Rates; models adversarial hardening as zero-sum game rather than cooperative MAS; does NOT target multi-agent delegation or tool-misuse fuzzing.
  - vs MASFuzzer: C7 (defense) — provides an automated defense-hardening loop; MASFuzzer's attack-side contribution (C1-C4) is complementary but must clarify it is not providing a defense mechanism.
  - https://arxiv.org/abs/2601.19726
- **Don't Trust Your Upstream: Exploiting LLM Multi-Agent System via Topology-Guided Adversarial Propagation (TOMA)** (arXiv 2512.04129, Dec 2024)
  - does: Black-box multi-hop attack that performs topology reconnaissance on MAS, then propagates adversarial payloads from exposed edge agents to high-privilege core agents across five representative topologies (star, chain, tree, mesh, hierarchy); optimizes the contamination propagation PATH through the topology using hierarchical payload encapsulation; achieves 40–78% ASR on Magentic-One, LangManus, OWL; does NOT target certainty signals or fuzzing coverage.
  - vs MASFuzzer: C4 (accountability-sink: edge agent is the entry point, core controller bears consequence) and C5 (real-frameworks: Magentic-One, LangManus, OWL); also threatens C1 (a topology-aware fuzzer vs a single-shot static attack)
  - https://arxiv.org/abs/2512.04129
- **Red-Teaming LLM Multi-Agent Systems via Communication Attacks (AiTM)** (ACL Findings 2025 / arXiv 2502.14847)
  - does: Introduces Agent-in-the-Middle (AiTM) attack that intercepts and manipulates inter-agent messages without compromising individual agents; an LLM-powered adversarial agent with a reflection mechanism generates contextually-aware malicious instructions inserted into the communication channel; black-box, operates at the MESSAGE layer rather than agent internals; optimizes message-level perturbations; evaluated on multiple frameworks and topologies including chain structures (>90% vulnerability).
  - vs MASFuzzer: C4 (accountability-sink: corrupting the channel rather than any single agent makes attribution ambiguous) and C1 (red-teaming vs. fuzzing coverage); contrasts with MASFuzzer's coverage-guided fuzzing of agent behavior rather than static message tampering
  - https://arxiv.org/abs/2502.14847
- **Infecting LLM Agents via Generalizable Adversarial Attack (MOI / Self-Propagating)** (NeurIPS 2024 (Contributed Talk) / OpenReview udsmFGMwlp)
  - does: Defines Multi-Agent One-Intervention (MOI) attacker that can influence only one agent in round 1; optimizes a self-propagating adversarial input such that infected agents repeat the payload to all neighbors with high probability; shows infection spreads to all agents in O(log N) steps; white-box optimization of the propagating string; does NOT exploit topology structure—propagation is topology-independent via shared memory/message passing.
  - vs MASFuzzer: C1 (coverage of agent states: MOI is a weaker attacker than MASFuzzer's coverage-guided approach) and C4 (single injection point, system-wide consequence = accountability-sink pattern)
  - https://openreview.net/forum?id=udsmFGMwlp
- **Agent Smith: A Single Image Can Jailbreak One Million Multimodal LLM Agents Exponentially Fast** (ICML 2024 / arXiv 2402.08567)
  - does: White-box adversarial image attack on multimodal MAS; a single crafted adversarial image jailbreaks one agent, which then spreads the image to all agents through shared memory in randomized pairwise chat; achieves full-system infection in O(log N) hops without further attacker intervention; optimizes universality of the adversarial IMAGE SIGNAL rather than a path; simulated up to one million LLaVA-1.5 agents; does NOT require topology knowledge.
  - vs MASFuzzer: C2 (continuum of attack severity: single-image to system-wide = continuous severity axis) and C4 (accountability-sink: initial compromised agent indistinguishable from later infected ones)
  - https://arxiv.org/abs/2402.08567
- **Prompt Infection: LLM-to-LLM Prompt Injection within Multi-Agent Systems** (arXiv 2410.07283, Oct 2024)
  - does: Demonstrates LLM-to-LLM prompt injection where a malicious prompt self-replicates across interconnected agents like a computer virus; a compromised agent spreads infection to downstream agents and coordinates data theft/scam/misinformation payloads; black-box setting; optimizes the SELF-REPLICATING PROMPT TEXT rather than a topological path; also shows infection propagates even when agents do not share all communications publicly; proposes LLM Tagging as defense.
  - vs MASFuzzer: C4 (accountability-sink: coordinated spread erases which agent was originally compromised) and C7 (defense: LLM Tagging baseline directly relevant to MASFuzzer's defense evaluation)
  - https://arxiv.org/abs/2410.07283
- **CORBA: Contagious Recursive Blocking Attacks on Multi-Agent Systems Based on Large Language Models** (arXiv 2502.14529, Feb 2025)
  - does: Injects a malicious prompt into an entry agent causing an infinite recursive blocking loop; contagiously propagates the DoS state to neighboring agents across arbitrary topologies (topology-independent); evaluated on AutoGen and CAMEL with commercial models; optimizes AVAILABILITY disruption rather than a jailbreak payload; does not need topology knowledge; contrast to TOMA which optimizes a specific propagation path toward high-privilege agents.
  - vs MASFuzzer: C1 (fuzzer vs. fixed-injection attack: CORBA uses single fixed entry point vs. MASFuzzer's coverage-guided exploration) and C4 (the accountability-sink problem persists—which agent caused the cascade?)
  - https://arxiv.org/abs/2502.14529
- **Attack the Messages, Not the Agents: A Multi-round Adaptive Stealthy Tampering Framework for LLM-MAS (MAST)** (AAAI 2025 / arXiv 2508.03125)
  - does: Uses Monte Carlo Tree Search + Direct Preference Optimization to train a policy model that generates multi-round message tampering strategies against LLM-MAS; operates on inter-agent MESSAGES without modifying agent internals; dual semantic and embedding-similarity constraints enforce stealthiness; black-box; optimizes a MESSAGE-TAMPERING POLICY over multiple rounds rather than a one-shot path signal.
  - vs MASFuzzer: C1 (policy-based optimizer vs. coverage-guided fuzzer: MAST optimizes for adaptive success, MASFuzzer optimizes for behavior coverage) and C3 (certainty-as-fitness: MAST's reward signal is success+stealthiness, not an uncertainty/certainty measure)
  - https://arxiv.org/abs/2508.03125
- **ClawWorm: Self-Propagating Attacks Across LLM Agent Ecosystems** (arXiv 2603.15727, Mar 2026)
  - does: First self-replicating worm against a production-scale open-source agent framework (OpenClaw, 40k+ instances); a single message hijacks the victim's persistent configuration (surviving session restart), executes an arbitrary payload, and propagates to every newly encountered peer with no further attacker intervention; evaluates three infection vectors and three payload types with real-execution verification; multi-hop experiments reveal LLM semantic degradation as a natural propagation constraint; no topology optimization.
  - vs MASFuzzer: C5 (real-frameworks: targets production OpenClaw deployment) and C4 (accountability-sink: persistent config hijack makes the infected agent appear to act autonomously)
  - https://arxiv.org/abs/2603.15727
- **SoK: Trust-Authorization Mismatch in LLM Agent Interactions** (arXiv 2512.06914, Dec 2025)
  - does: Systematization-of-Knowledge paper surveying 200+ papers on LLM agent security; proposes the Belief-Intention-Permission (B-I-P) framework showing that diverse attacks (prompt injection, tool poisoning, inter-agent manipulation) share a common root: static permissions decoupled from runtime trustworthiness; does NOT present a new attack but provides a unified threat taxonomy covering trust hierarchies and authorization mismatches in multi-agent topologies.
  - vs MASFuzzer: C4 (accountability-sink: the B-I-P framework formalizes why inter-agent trust mismatches create attribution gaps) and C6 (independent oracle: the SoK's taxonomy is a natural reference for MASFuzzer's oracle design for what constitutes a policy violation)
  - https://arxiv.org/abs/2512.06914
- **To trust or not to trust: Attention-based Trust Management for LLM Multi-Agent Systems** (arXiv 2506.02546, June 2025)
  - does: Proposes A-Trust, a lightweight method that reads internal LLM attention heads across six orthogonal trust dimensions to score upstream message trustworthiness without external prompts or verifiers; builds a full Trust Management System (TMS) enabling both message-level and agent-level trust calibration in multi-agent settings. White-box (attention access required). Directly optimizes for discounting untrustworthy upstream agent confidence.
  - vs MASFuzzer: C7 defense — closest existing work to certainty/trust discounting of upstream agents; also relevant to C4 (accountability-sink) since it attributes trust per agent.
  - https://arxiv.org/abs/2506.02546
- **SentinelNet: Safeguarding Multi-Agent Collaboration Through Credit-Based Dynamic Threat Detection** (ACM Web Conference (WWW) 2026, arXiv 2510.16219)
  - does: Decentralized framework giving each agent a credit-based detector (trained via contrastive learning on augmented adversarial debate trajectories) that autonomously scores neighbor message credibility and applies bottom-k neighbor elimination to suppress malicious communications; achieves ~100% malicious-agent detection within two debate rounds. Black-box to peer agents. Optimizes for both detection accuracy and system task recovery (utility-aware).
  - vs MASFuzzer: C7 defense — utility-aware mitigation that discounts low-credit upstream agents; most directly relevant to the certainty-as-fitness and accountability-sink claims.
  - https://dl.acm.org/doi/10.1145/3774904.3792462
- **AgentSafe: Safeguarding Large Language Model-based Multi-agent Systems via Hierarchical Data Management** (arXiv 2503.04392, March 2025)
  - does: Introduces ThreatSieve (verifies information authority and blocks impersonation in inter-agent messages) and HierarCache (adaptive memory management preventing unauthorized access and memory poisoning); classifies information by security levels and restricts access to authorized agents. Black-box external wrapper. Optimizes for defense success rate (>80%) under adversarial conditions. First systematic defense for agent memory.
  - vs MASFuzzer: C7 defense — message provenance and sanitization; addresses C4 (accountability-sink) via authority-based information routing.
  - https://arxiv.org/abs/2503.04392
- **G-Safeguard: A Topology-Guided Security Lens and Treatment on LLM-based Multi-agent Systems** (ACL 2025 (Main), arXiv 2502.11127)
  - does: Builds a multi-agent utterance graph, applies GNN-based anomaly detection on node/edge features encoding agent interaction history, then performs topological intervention (edge pruning) to remove malicious information pathways; recovers >40% performance under prompt injection. Black-box to individual agents (graph-level monitor). Optimizes for task performance recovery after attack.
  - vs MASFuzzer: C7 defense — verifier/oracle hardening via graph topology; relevant to C6 (independent oracle) since the GNN monitor is external to the task agents.
  - https://arxiv.org/abs/2502.11127
- **MAS-Shield: A Defense Framework for Secure and Efficient LLM MAS** (arXiv 2511.22924, November 2025)
  - does: Coarse-to-fine filtering pipeline: (1) Critical Agent Selection targets high-influence nodes; (2) lightweight sentry models rapidly filter benign cases; (3) escalates ambiguous signals to a heavyweight committee for global consensus auditing. Addresses the dilemma between single-auditor (cheap but fragile) and committee (robust but costly) defenses. Multi-agent, black-box compatible. Optimizes for both security and compute efficiency.
  - vs MASFuzzer: C7 defense — verdict hardening via committee consensus; addresses C4 (accountability-sink) by targeting high-influence nodes; utility-aware resource allocation.
  - https://arxiv.org/abs/2511.22924
- **INFA-Guard: Mitigating Malicious Propagation via Infection-Aware Safeguarding in LLM-Based Multi-Agent Systems** (arXiv 2601.14667, January 2026)
  - does: Explicitly distinguishes three agent states (benign, attacker, infected-benign) rather than a binary benign/malicious split; uses infection-aware detection with topological constraints to localize attack sources and propagation ranges, then rehabilitates infected agents while replacing confirmed attackers. Reduces ASR by average 33% with cross-model robustness. Optimizes for propagation containment with topological integrity preservation.
  - vs MASFuzzer: C7 defense — infection-state taxonomy directly bears on the accountability-sink problem (C4); propagation modeling complements C3 (certainty-as-fitness) by tracking how manipulated confidence spreads.
  - https://arxiv.org/abs/2601.14667
- **GroupGuard: A Framework for Modeling and Defending Collusive Attacks in Multi-Agent Systems** (arXiv 2603.13940, March 2026)
  - does: Models coordinated multi-agent collusion via sociological strategies (agents use group consensus signals to manipulate the manager/judge); defends with continuous graph-based monitoring, active honeypot inducement, and structural pruning to detect and isolate colluding agents. Detection accuracy up to 88% across five datasets and four topologies. Multi-agent system, black-box to individual colluders.
  - vs MASFuzzer: C7 defense — most directly relevant to the 三人成虎 (multi-source consensus manipulation) attack vector; also bears on C4 (accountability-sink) and C3 (certainty-as-fitness) since colluders exploit the judge's trust in group verdicts.
  - https://arxiv.org/abs/2603.13940
- **Monitoring LLM-based Multi-Agent Systems Against Corruptions via Node Evaluation** (arXiv 2510.19420, October 2025)
  - does: Reconstructs MAS as a DAG, computes per-agent contribution scores via backward propagation from the final decision, identifies latent malicious agents by contribution anomaly, and repairs the system by removing their influence. Dynamic defense paradigm (continuous re-evaluation) vs. static graph defense. Black-box monitor. Optimizes for post-corruption decision quality.
  - vs MASFuzzer: C7 defense — independent oracle for verdict attribution (C6); contribution-score attribution is a form of upstream confidence discounting directly relevant to C3 (certainty-as-fitness).
  - https://arxiv.org/abs/2510.19420
- **Explainable and Fine-Grained Safeguarding of LLM Multi-Agent Systems via Bi-Level Graph Anomaly Detection (XG-Guard)** (arXiv 2512.18733, December 2025)
  - does: Proposes XG-Guard, a bi-level agent encoder jointly modeling sentence-level and token-level representations of agent messages for anomalous agent detection; adds explainability layer so the system can attribute which specific lexical cues triggered anomaly flags. Addresses coarse-grain limitation of prior GNN-based defenses. Black-box monitor over the MAS.
  - vs MASFuzzer: C7 defense — explainable verifier hardening; fine-grained message provenance tracing relevant to C4 (accountability-sink) and C6 (independent oracle).
  - https://arxiv.org/abs/2512.18733
- **Ferret: Faster and Effective Automated Red Teaming with Reward-Based Scoring Technique** (arXiv 2408.10701, 2024)
  - does: Black-box evolutionary red-teaming that extends Rainbow Teaming by generating multiple adversarial prompt mutations per iteration and ranking/selecting them with a learned reward model (or Llama Guard / LLM-as-judge) as a dense fitness signal instead of waiting for a sparse binary jailbreak outcome; single-target, single-agent; optimizes expected harm score of the next prompt.
  - vs MASFuzzer: C3 (certainty-as-fitness): Ferret is the closest prior work to using a learned mediator/reward signal as dense fitness for attack generation. It shows a reward model used as a surrogate fitness outperforms binary-outcome search by +46% ASR, directly pre-empting the claim that a learned mediator provides novel guidance. However, its reward model measures predicted harm, NOT the attacker's internal certainty/confidence about the victim, which is the MASFuzzer-specific wedge.
  - https://arxiv.org/abs/2408.10701
- **Rainbow Teaming: Open-Ended Generation of Diverse Adversarial Prompts** (arXiv 2402.16822, 2024 (Meta FAIR))
  - does: Black-box quality-diversity (MAP-Elites) search over a 2D behavioral descriptor space (risk category x attack style) to populate an archive of adversarial prompts against a target LLM; mutations are made by an LLM mutator; fitness is binary judge success; single-agent, iterative archive.
  - vs MASFuzzer: C1 (fuzzer): Rainbow Teaming is the primary prior fuzzer baseline for evolutionary red-teaming. It optimizes diversity+effectiveness but uses a binary oracle (C6), has no learned mediator signal (not C3), and is single-agent (not C4). MASFuzzer's key delta must be the certainty-as-mediator fitness and multi-agent coverage.
  - https://arxiv.org/abs/2402.16822
- **RainbowPlus: Enhancing Adversarial Prompt Generation via Evolutionary Quality-Diversity Search** (arXiv 2504.15047, 2025)
  - does: Extends Rainbow Teaming with a richer evolutionary operator set and co-evolving niches; black-box; single-agent; optimizes QD coverage and ASR; achieves 81.1% ASR 9x faster than AutoDAN-Turbo and generates 100x more unique prompts (e.g., 10,418 vs. 100); uses binary judge as fitness, not a learned mediator.
  - vs MASFuzzer: C1 (fuzzer) / C2 (continuum): RainbowPlus is the strongest direct evolutionary QD competitor to MASFuzzer's search loop. It does NOT use a causal mediator or certainty signal (C3), and is single-agent (C4 absent). Must be beaten or distinguished on mediator-guided density.
  - https://arxiv.org/abs/2504.15047
- **Learning Diverse Attacks on Large Language Models for Robust Red-Teaming and Safety Tuning** (ICLR 2025 (arXiv 2405.18540))
  - does: Trains an attacker LLM with GFlowNet fine-tuning to sample from a diverse distribution of effective attack prompts against a target LLM; uses a toxicity/harm reward model as distribution target; a secondary smoothing phase prevents mode collapse; black-box at inference, white-box attacker training; single-agent.
  - vs MASFuzzer: C1 (fuzzer) / C3 (certainty-as-fitness): GFlowNet red-teaming uses a learned harm-reward model as the shaping signal for the attacker's distribution—structurally the closest to using a learned mediator as fitness. Key differences from MASFuzzer: the mediator here is harm probability (not victim's internal certainty), training is offline, and there is no multi-agent accountability-sink structure (C4).
  - https://arxiv.org/abs/2405.18540
- **Diverse and Effective Red Teaming with Auto-generated Rewards and Multi-step Reinforcement Learning** (arXiv 2412.18693, 2024 (OpenAI))
  - does: Auto-generates per-goal rule-based rewards (RBRs) for diverse attacker goals, then trains an attacker LLM with multi-step RL where diversity bonuses reward prompts differing from past attempts; black-box target, white-box attacker training; multi-step but single agent per goal; used for prompt injection and unsafe-response elicitation.
  - vs MASFuzzer: C1 (fuzzer) / C3 (certainty-as-fitness): Auto-generated per-goal rewards act as learned surrogate fitness signals—closest prior work from a top lab to the mediator-as-fitness concept. Does NOT use victim certainty as the mediator; rewards are rule-based graders over outputs, not internal model states. No multi-agent accountability sink (C4).
  - https://arxiv.org/abs/2412.18693
- **PAL: Proxy-Guided Black-Box Attack on Large Language Models** (arXiv 2402.09674, 2024)
  - does: Proxy-guided black-box adversarial trigger optimization: uses a white-box surrogate LLM to estimate gradients and guide optimization of adversarial suffixes against a black-box target LLM API; query-based; single-agent; optimizes surrogate loss as a mediator for the sparse final API outcome; achieves 84% ASR on GPT-3.5-Turbo.
  - vs MASFuzzer: C3 (certainty-as-fitness): PAL is the clearest example of using a learned/proxy mediator (surrogate model logits) instead of sparse final outcome as the optimization signal for LLM attack generation. However, it operates on token-level suffix search (not semantic/conversational), targets single-turn completion, and is not multi-agent (C4 absent).
  - https://arxiv.org/abs/2402.09674
- **TROJail: Trajectory-Level Optimization for Multi-Turn Large Language Model Jailbreaks with Process Rewards** (arXiv 2512.07761, 2024/2025)
  - does: Multi-turn RL attacker trained with two process reward signals as dense intermediates: (1) a penalty for overly harmful prompts that trigger refusals early, (2) a semantic-relevance reward steering intermediate responses toward the harm target; outcome reward is final-turn harmfulness; multi-turn single-agent; evaluated on HarmBench/StrongREJECT.
  - vs MASFuzzer: C3 (certainty-as-fitness) / C2 (continuum): TROJail explicitly introduces learned process/intermediate rewards as dense fitness signals to replace sparse final-outcome supervision—the structural analog to MASFuzzer's certainty-as-mediator. Key difference: mediators are semantic relevance + refusal penalty, not victim model confidence/certainty; and attack is against a single LLM, not a multi-agent system (C4, C5 absent).
  - https://arxiv.org/abs/2512.07761
- **Stable-GFlowNet: Toward Diverse and Robust LLM Red-Teaming via Contrastive Trajectory Balance** (arXiv 2605.00553, 2025/2026)
  - does: Improves GFlowNet-based red-teaming by eliminating partition-function estimation through pairwise contrastive trajectory balance, adding a fluency stabilizer to prevent gibberish local optima, and applying robust masking against noisy rewards; black-box target; attacker trained to sample diverse harmful prompts proportional to a harm reward model; single-agent.
  - vs MASFuzzer: C3 (certainty-as-fitness): Like the ICLR-2025 GFlowNet paper, this uses a harm reward model as the learned distribution target (mediator). The novelty vs. MASFuzzer is the same gap: mediator = harm probability not victim certainty; no multi-agent structure; no accountability-sink or independent-oracle concerns (C4, C6 absent).
  - https://arxiv.org/abs/2605.00553
- **Jailbreaking as a Reward Misspecification Problem** (arXiv 2406.14393, ICLR 2025)
  - does: Frames LLM vulnerability as reward misspecification during RLHF alignment; introduces ReGap metric to quantify misspecification; builds ReMiss, a system that searches adversarial prompts in the reward-misspecified space where the alignment reward incorrectly ranks harmful responses as safe; white-box reward model access; single-agent; state-of-the-art on AdvBench.
  - vs MASFuzzer: C3 (certainty-as-fitness): ReMiss explicitly exploits the victim's internal reward model (a causal mediator of alignment) as the search fitness—the most mechanistically direct use of a mediator signal for attack generation. However, it requires white-box access to the victim's reward model (not black-box), operates on single-turn settings, and has no multi-agent structure (C4, C5 absent).
  - https://arxiv.org/abs/2406.14393
- **When Persuasion Overrides Truth in Multi-Agent LLM Debates: Introducing a Confidence-Weighted Persuasion Override Rate (CW-POR)** (arXiv 2504.00374, April 2025)
  - does: Black-box single-turn multi-agent debate study: one agent states a truthful answer (TruthfulQA), an adversary defends a falsehood, and the same LLM architecture judges — authors introduce CW-POR to measure how often (and with what confidence) the judge selects the false claim; finds even small models crafting verbose, assertive arguments override truth at high judge confidence.
  - vs MASFuzzer: C3 (certainty-as-fitness): most direct precursor — shows expressed confidence of the adversarial argument, not factual correctness, drives judge flips; directly motivates using certainty as a search signal in a fuzzer.
